pip install abc
error: externally-managed-environment

October 27, 2024

In Linux or MacOS, you might have stumbled upon this error some times. This basically occurs when you install all packages in global Python. So it can be avoided with the use of virtual environments. But even still in rare cases it can be present. And few tries later when you still cannot solve it, you might want to delete python out of frustation. So lets discuss how to troubleshoot and solve it. These steps are basic and you shall follow them to troubleshoot any python related problems.

Steps to follow for troublshooting:

  • check python version you are using right now. Use python3 -V or python3 --version. [## python3 for MAC OS and python in windows]
  • Check all installed pythons in your device. Use which -a python3.x Remember that in MAC OS the python installed in /usr/bin/ is the preinstalled python3.9, So dont tamper it.
  • check PATH env variable. Use echo $PATH. Now look at the python path mentioned there.
  • Use pyenv for python versions management. More about pyenv here. You need homebrew for this so install it using: cmd1 or cmd2
  • Check configuration files like ~/.zshrc, ~/.zprofile. and edit the PATH variable. Add this line there: 
                    
                # Add pyenv to PATH
                export PATH="$HOME/.pyenv/bin:$PATH"
                eval "$(pyenv init --path)"
                eval "$(pyenv init -)"
  • Add Homebrew to PATH if not already. 
                    
                  # Add Homebrew to PATH
                  export PATH="/opt/homebrew/bin:/opt/homebrew/sbin:$PATH"
                  # Homebrew initialisation
                  eval "$(/opt/homebrew/bin/brew shellenv)"
  • save your changes and use source ~/.zshrc or source ~/.zprofile
